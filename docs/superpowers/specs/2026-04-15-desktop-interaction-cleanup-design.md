# Desktop Interaction Cleanup Design

Date: 2026-04-15
Branch: `codex/local-first-desktop-delivery`

## Goal

把当前桌面化主线里最不一致的 3 个页面收口到统一的交互模型：

- `/reviews/new` 变成只负责创建评审的单栏主流程页
- `/models` 变成和规则管理一致的列表中心页
- `/docs` 变成统一的文档入口，采用经典三栏知识库布局

本轮目标是修正页面交互逻辑仍偏网页后台的问题，不包含桌面打包启动链路修复，不扩展到首页、评审详情页或其它页面重设计。

- 重构 [app/models/page.tsx](/Users/jiangdongzhe/Dev/ai-project/plreview/app/models/page.tsx) 和 [components/model-manager.tsx](/Users/jiangdongzhe/Dev/ai-project/plreview/components/model-manager.tsx) 的交互形态

## Approach

采用“一致性优先改造”策略：

1. 让操作页只承载操作，不承载长说明
2. 让配置管理页统一采用“列表主视图 + 搜索 + 新增按钮 + 右侧编辑抽屉”
3. 让文档页统一承载流程说明，并提供桌面工具常见的双侧目录导航

这个方案优先解决认知负担和交互割裂问题，不额外扩展范围。

## Information Architecture

### `/reviews/new`

- 页面只负责创建评审
- 页面采用单栏主流程
- 页面不展示流程说明、操作教程或大段解释文案
- 文案只服务于当前动作
- 操作顺序固定为：
  - 批次信息
  - 模型选择
  - 规则选择
  - 文件导入与解析结果
  - 提交

### `/models`

- 页面职责是“管理模型配置”，不是“填写长表单”
- 首屏结构固定为：
  - 页面标题与简短摘要
  - 指标卡片
  - 搜索栏
  - 新增按钮
  - 列表表格
- 新增和编辑都通过右侧抽屉完成
- 页面顶部现有的内嵌表单完全移除

### `/docs`

- 页面是统一文档入口
- 采用经典三栏布局：
  - 左侧文档目录
  - 中间正文
  - 右侧文章内目录
- 左侧负责在主题文档之间切换
- 右侧负责当前文章的章节锚点导航
- 左右侧栏都支持折叠

## Page Design

### Reviews New

#### Layout

- 顶部保留页面标题和极短上下文提示
- 主体为纵向单栏，不再使用 50/50 双栏主布局
- 各段之间使用清晰的区块层次，但不再并排展示说明区

#### Interaction

- 保留当前桌面工作台的核心能力：
  - 导入本地文件
  - 展示解析状态
  - 查看文件摘要
  - 删除单个文件
  - 清空失败项
  - 清空当前工作区
- 提交按钮只有在以下条件都满足时可用：
  - 批次名称非空
  - 已选择模型配置
  - 已选择至少一条规则
  - 至少存在一个可提交文件
- 提交成功后直接跳转到 `/reviews`

#### Content Rules

- 不保留流程说明、帮助文档摘录或建设期文案
- 如果用户缺少模型配置，只显示与当前动作直接相关的引导，例如前往模型页

### Models

#### Layout

- 结构与规则管理页对齐
- 列表优先，编辑其次
- 列表展示核心字段：
  - 配置名称
  - 供应商
  - 运行模式
  - 默认模型
  - Key 状态
  - 启用状态

#### Interaction

- 顶部提供搜索输入和“新增模型”按钮
- 行内操作固定为：
  - 编辑
  - 启用/停用
  - 删除
- 新增与编辑共用右侧抽屉
- 抽屉关闭后回到原列表上下文，不触发页面跳转

#### Drawer

- 抽屉承载完整模型表单
- 表单字段沿用当前已有能力，不在本轮删减业务字段
- 如果字段较多，抽屉内部允许滚动，但页面列表保持稳定

### Docs

#### Layout

- 桌面端采用三栏布局
- 左侧栏宽度用于承载文档目录
- 中间正文是视觉中心
- 右侧栏宽度用于承载文章内结构目录

#### Navigation

- 左栏展示主题列表，例如：
  - 工作流程
  - 模型配置
  - 规则编写
  - 结果阅读
- 右栏展示当前主题的章节锚点
- 切换左栏主题时：
  - 中间正文同步切换
  - 右栏目录同步刷新

#### Collapse Behavior

- 左右侧栏都支持折叠
- 折叠后保留明确的展开入口，不允许“隐藏后无处找回”
- 折叠状态是局部 UI 状态，不影响文档内容

#### Responsive Behavior

- 中等宽度下允许用户主动折叠一侧或两侧
- 窄屏下不强制保留三栏
- 窄屏降级为正文优先布局：
  - 左目录通过顶部入口打开
  - 右目录通过当前文章目录入口打开

## State And Messaging

### Reviews New

- 空状态只提示当前缺少的操作条件
- 错误提示尽量贴近触发区域
- 文件解析失败应落在文件区内展示，不放大为全页说明

### Models

- 空状态引导用户新增第一个模型配置
- 删除操作应有明确确认
- 启停属于轻操作，应在列表内直接完成

### Docs

- 如果某一主题没有足够多的章节层级，右栏可简化或隐藏
- 文档页承担说明职责，其它业务页不再重复长说明

## Visual And Content Principles

- 统一使用“文档”命名，不混用“帮助”“Help”
- 操作页优先保证主任务清晰，不被说明内容挤压
- 桌面端布局应更接近知识库和工具型应用，而不是营销型网页
- 视觉重点始终在当前任务或当前正文，不在辅助说明区

## Testing Strategy

- 更新或新增组件测试，覆盖：
  - `新建评审` 单栏主流程的关键控件和可提交条件
  - `模型管理` 的搜索、抽屉打开关闭、空状态和行内操作
  - `文档` 页的双侧目录渲染、主题切换和折叠逻辑
- 不在本轮引入桌面打包端到端测试

## Risks

- `新建评审` 当前组件已承载较多工作台逻辑，改为单栏时要避免打乱已有文件导入行为
- `模型管理` 从内嵌表单切换为抽屉后，表单状态和删除确认要设计清楚，避免误操作
- `文档` 页三栏布局如果样式边界处理不好，容易在中等宽度下挤压正文

## Success Criteria

- `新建评审` 页面不再出现说明区与操作区并排的双栏主结构
- `模型管理` 页面首屏不再出现顶部内嵌大表单
- `文档` 页具备左文档目录、中文章正文、右文章目录的三栏桌面布局
- 左右目录都可以折叠且可恢复
- 这 3 个页面的交互风格明显更统一

## 2026-04-16 Workflow Correction

### Background

- 在桌面入口已经切换为“选择本地文件”的前提下，`新建评审` 页仍残留网页回退时期的数据模型和文案
- 页面会继续展示：
  - `可提交 0 条`
  - `待从桌面导入`
  - `重新导入`
  - `清理待重新导入`
- 这些状态来自浏览器回退占位行，而不是来自真实已导入文档，因此会让桌面流程看起来像“选择 + 导入 + 提交”的三段式

### Correction Decision

- `新建评审` 的桌面 workflow 收敛为两步：
  - 选择本地文件
  - 开始评审
- 桌面环境中，选择文件后必须立即完成本地导入与解析，工作台只展示真实导入结果
- 用户界面不再暴露浏览器回退占位概念，不再出现“待从桌面导入”或“重新导入”之类的中间态

### Updated Reviews New Rules

- Step 3 的文件区域只展示真实已导入文件
- 统计文案统一改为“已导入”或“待评审”语义，不再使用“可提交/待重新导入”双轨统计
- 表格不再提供：
  - 状态筛选
  - 待重新导入统计
  - 重新导入操作
  - 清理待重新导入操作
- 浏览器环境不再生成伪导入记录
- 如果当前不具备桌面桥接能力，文件区域只显示阻断提示，明确要求在桌面应用中启动后再导入

### Testing Update

- 组件测试需要覆盖：
  - 桌面环境中点击一次“选择本地文件”后立即写入真实导入行
  - 页面不再渲染“待从桌面导入”“重新导入”“清理待重新导入”“可提交 X 条”一类旧语义
  - 非桌面环境下显示阻断提示，而不是生成占位工作台数据
