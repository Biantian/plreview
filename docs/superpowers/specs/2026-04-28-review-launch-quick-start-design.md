# 新建批次快启动改版设计

日期：2026-04-28

## 背景

当前“新建批次”页面同时承载了批次配置、规则浏览、文件工作台、提交摘要等多种职责，页面更像控制台拼盘，而不是高频启动流程。随着规则数量增长，当前页内直接铺开规则列表的方式会继续恶化信息密度与扫描成本。

这次改版的目标不是局部润色，而是重新定义“开始批次”这条链路里的页面职责：

- `工作台` 负责判断是否可以开工，并提供主入口。
- `新建批次` 是唯一的批次创建入口。
- `规则库` 只负责规则资产管理，不参与当前批次配置。
- `模型配置` 只负责模型资产管理，不承担前置流程职责。
- `评审任务` 只负责查看、处理与导出任务。

## 用户目标

用户希望尽快开始一个新批次，同时在规则变多后仍能快速确认“这次到底会按哪些规则跑”。用户不希望被重复解释默认行为，也不希望为了配置本次批次在多个页面间跳转。

## 设计原则

1. `新建批次` 必须是一条顺序明确的快启动流程。
2. 规则选择的复杂度收进抽屉，主页面只展示本次选择结果。
3. 默认行为可以轻提示一次，但不反复教育用户。
4. 规则库不再承担任何“当前批次配置”语义。
5. 页面主信息优先展示“本次会发生什么”，而不是低价值统计。

## 页面职责重排

### 1. 工作台 `/`

工作台退回概览页，只承担两件事：

- 显示当前是否具备开始批次的条件。
- 提供主按钮 `开始新批次`。

首页保留的状态信息：

- 是否存在可用模型配置。
- 是否存在启用规则。
- 最近评审记录。

首页不再提前承载规则选择、批次配置、或“带规则进入批次”之类动作。缺模型或缺规则时，只给出明确状态提示和对应维护页面入口。

### 2. 新建批次 `/reviews/new`

这是唯一的创建入口，承担完整配置与启动流程：

- 批次名称
- 模型选择
- 默认带入上次批次规则
- 打开规则抽屉重选
- 文件导入
- 启动批次

这页的核心是“快启动”，而不是“工作台控制台”。

### 3. 规则库 `/rules`

规则库只做资产管理：

- 新增、编辑、启停、删除
- 搜索规则

不再出现任何将规则带入当前批次的动作，不承担流程推进职责。

### 4. 模型配置 `/models`

模型配置页只做模型维护和启用状态管理。允许提供轻量回流到“新建批次”的入口，但不承载本次批次的具体选择语义。

### 5. 评审任务 `/reviews`

评审任务页继续承担任务列表、状态查看、重试、导出等后处理能力。顶部保留 `新建批次` 入口，但不增加任何创建前配置逻辑。

## 新建批次页面重排

页面改为单列优先阅读流，严格按照创建顺序组织。

### 区块 1：基础信息

仅保留：

- 批次名称
- 模型选择

这里的目标是让用户快速确认“本次用哪个模型”。页面不再通过多组 pills 重复展示模型状态，也不强调资产管理信息。

### 区块 2：规则摘要

这是页面的核心区块。

默认行为：

- 页面进入时，自动带入“最近一次批次使用的规则”。
- 页面顶部仅使用一个轻量消息提示，例如 `已带入上次批次规则`。
- 该提示只在首次进入页面时出现一次，几秒后自动消失，可手动关闭。

主操作：

- `选择规则`
- `一键清空`

主页面展示的是“本次会提交哪些规则”，而不是“规则总数”或“已选/总数”。

规则摘要使用精选规则卡片展示，每张卡片包含：

- 规则名称
- 一行简述
- 分类
- 严重级别
- 单条移除动作

当已选规则较多时：

- 默认展示前 3 至 5 条
- 提供 `查看全部 N 条` 展开
- 展开后仍为卡片流，而不是表格

### 区块 3：文件导入

未导入文件时，重点展示上传动作；导入后再展示导入结果列表。

空态不再渲染大面积表格骨架。设计目标是：

- 空态强调 `选择本地文件`
- 导入后强调文件结果与状态

### 区块 4：启动区

启动区只承担最后确认与提交职责，保留：

- 当前是否可启动的简短状态
- 主按钮 `开始评审`
- 缺失项提示

不再重复渲染一整套“批次 / 规则 / 文件”的统计快照。前面已经确认过的信息，不应在这里再次复制。

## 规则抽屉设计

规则抽屉是“为当前批次挑规则”的专用面板，不是规则库缩小版。

### 抽屉结构

#### 顶部操作区

仅保留必要操作：

- 搜索框
- `一键清空`
- `恢复上次`

不再添加解释性说明文案，不重复解释默认带入逻辑。

#### 中部结果区

展示规则搜索结果与当前勾选状态。

每条结果包含：

- 规则名称
- 一行简述
- 分类
- 严重级别
- 勾选状态

整行可点击切换是否加入本次批次。

#### 底部确认区

固定展示：

- 当前已选数量
- `取消`
- `确认带回`

### 交互模型

抽屉内使用暂存态：

- 打开抽屉后，用户可自由搜索、勾选、清空、恢复
- `取消` 关闭抽屉并丢弃本次改动
- `确认带回` 才同步到主页面

这样用户可以放心试选，不会因为临时操作污染当前批次配置。

## 搜索与模糊匹配

### 搜索范围

搜索命中字段：

- 规则名称
- 分类
- 描述
- 严重级别文案

### 文本归一化

搜索前统一进行：

- 去首尾空格
- 合并连续空格
- 忽略大小写

### 匹配优先级

匹配结果按以下优先级排序：

1. 名称完整命中
2. 名称部分命中
3. 分类命中
4. 描述命中
5. 严重级别命中

当用户输入多个词时，同时命中多个词的结果优先，而不是只要命中其中一个词就排前。

### 结果排序

同等匹配下，再按下列顺序排序：

1. 当前已选规则优先
2. 上次批次使用过的规则优先
3. 其他规则保持稳定原顺序

### 首版不做

首版明确不包含：

- 拼音搜索
- 拼音首字母搜索
- 同义词扩展
- 复杂高亮解释
- 高级多维筛选面板

首版只需要把中文关键词检索与相关度排序做好。

## 状态与反馈

### 首次进入页面

- 如果成功带入上次批次规则，显示一次顶部轻提示。
- 不在规则区和抽屉里重复解释同一事实。

### 搜索无结果

只显示简短反馈，例如：

- `没有匹配的规则，换个关键词试试`

### 清空与恢复

- `一键清空` 立即清除当前抽屉暂存选择。
- `恢复上次` 将抽屉暂存选择恢复为“上次批次规则集”。

## 数据与行为要求

1. 系统需要记录“最近一次批次使用的规则集合”。
2. 新建批次页加载时优先读取该集合并作为默认选中项。
3. 如果没有历史批次，则规则默认不选中。
4. 抽屉搜索结果必须与主页面摘要使用同一份规则数据源。
5. 主页面与抽屉之间通过显式确认同步，避免半成品状态污染页面。

## 不在本次范围

- 重做全局导航信息架构
- 改变规则库的资产管理定位
- 把模型配置页改成流程页
- 为规则库新增“加入当前批次”能力
- 为首页增加批次配置能力

## 验收标准

1. 用户可以从工作台通过单一主入口进入新建批次。
2. 新建批次页打开时，若存在历史批次，则默认带入上次批次规则，并只提示一次。
3. 主页面规则区显示已选规则摘要卡片，而不是规则总量统计。
4. 用户可通过抽屉搜索并调整本次规则集合。
5. 抽屉支持 `一键清空` 与 `恢复上次`。
6. 抽屉搜索结果按相关度优先，不再只是简单包含过滤。
7. 规则库页不承担任何当前批次配置职责。
8. 工作台页不承担规则选择或批次配置职责。
