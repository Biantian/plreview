# 布局与信息层级重排设计

## 1. 背景

当前版本虽然已经把帮助文案从多个页面收拢了一部分，但页面结构仍然有几个明显问题：

- 多个业务页继续沿用左右二分布局，导致静态说明区长期占据大块版面。
- 一些页面右侧内容固定且很短，左侧内容会随着使用增长，视觉重心明显失衡。
- 首页仍保留了较多“设计理念式”自我描述，而不是直接服务当前工作。
- 顶部导航缺少当前页高亮，页面定位感不足。
- 评审详情中的“原文命中”排版间隔偏大，正文阅读节奏不自然。

这轮重排的目标不是继续增加说明文本，而是重新定义哪些区域是“增长型工作区”，哪些区域只是“短暂提示”，并把静态说明降到最低注意力层级。

## 2. 设计目标

### 2.1 目标

- 首页改成真正的操作台，不再出现“设计原则”或“页面理念”类自我介绍文本。
- 除评审详情页外，其余页面默认收敛成单栏主工作区。
- 双栏只保留在确实需要并行阅读的页面，并采用非对称比例，而不是机械 50/50。
- 帮助信息改成短提示、入口链接或集中式文档页，不再长期占据半个屏幕。
- 顶部导航新增当前页高亮，提升定位感。
- 评审详情页优化正文阅读节奏，压缩不必要的段间距。

### 2.2 非目标

- 本轮不做视觉语言大换血，仍沿用现有卡片、按钮、色彩和整体品牌调性。
- 本轮不引入复杂的抽屉系统、悬浮帮助系统或新的状态管理。
- 本轮不重做评审结果的数据结构，只处理布局和信息呈现方式。

## 3. 页面结构原则

### 3.1 区分增长型区域与静态区域

页面内容按两类看待：

- 增长型区域：会随着业务使用不断增长，例如任务列表、规则列表、模型配置列表、原文块、问题清单。
- 静态区域：说明文本、阅读提示、当前页介绍、帮助入口。

布局上优先给增长型区域足够宽度和连续空间，静态区域只保留最短说明，并尽量放到：

- 页头下一句说明
- 小型提示卡
- 帮助页链接

而不是长期保留一个大侧栏。

### 3.2 单栏优先

以下页面统一改为单栏：

- 首页
- 评审列表
- 新建评审
- 规则管理
- 模型设置
- 帮助中心

原因：

- 这些页面的核心任务都只有一个主工作区。
- 目前的右侧说明内容不足以支撑半屏存在。
- 单栏会让列表、表单和卡片在桌面端更集中，也更适合后续内容增长。

### 3.3 非对称双栏只用于并行阅读

只有评审详情页保留双栏，因为它天然存在并行阅读需求：

- 左侧：原文命中与上下文
- 右侧：问题清单与当前问题详情

这一页改为非对称双栏，例如：

- 桌面端：`minmax(0, 1.75fr) minmax(320px, 0.95fr)`
- 较窄桌面：进一步收敛到 `1.5fr / 1fr`
- 移动端：直接折叠为单栏

## 4. 方案对比

### 4.1 方案 A：全面单栏，详情页也改单栏

优点：

- 最简单。
- 一致性最高。

缺点：

- 详情页会丢掉“边看原文边看问题”的效率。

### 4.2 方案 B：单栏业务页 + 非对称详情页

优点：

- 业务页更干净，主任务更集中。
- 详情页仍保留并行阅读优势。
- 最符合当前内容形态。

缺点：

- 需要额外调一轮详情页栅格和间距。

### 4.3 方案 C：保留双栏，但按页面动态改比例

优点：

- 视觉形式更统一。

缺点：

- 即使不是 50/50，很多页面依然会保留“说明侧栏常驻”的问题。
- 会继续浪费桌面宽度。

### 4.4 推荐方案

选择方案 B：单栏业务页 + 非对称详情页。

原因：

- 最直接解决你指出的“静态区域抢版面”问题。
- 保留详情页真正有价值的双栏阅读能力。
- 不需要引入复杂交互，也便于在现有代码基础上快速落地。

## 5. 页面级设计

### 5.1 首页

首页定位改成“操作台”，只保留与当前工作直接相关的信息：

- 最近任务
- 快捷入口
- 模型与规则状态摘要

移除内容：

- “首页保留流程总览”“设计原则”“为什么这样组织页面”等自我解释型文案
- 对页面布局理念的介绍

保留一条非常短的说明即可，例如“从这里进入任务、规则和模型配置”。

### 5.2 评审列表

改为单栏：

- 顶部页头
- 状态统计
- 任务列表

删除当前右侧的大块“阅读节奏”和“补充说明”。

替代方式：

- 在标题下保留一句说明
- 列表上方放一个很短的帮助链接，例如“查看结果阅读说明”

### 5.3 新建评审

改为单栏表单页：

- 页头
- 任务设置
- 文档导入
- 规则选择
- 提交按钮

“快速提示”和“当前资源”不再作为右侧两张卡常驻存在。

替代方式：

- 模型选择器内直接显示配置状态
- 页面顶部保留一个短帮助入口
- 可在提交按钮附近补一行极短提示，而不是独立半屏侧栏

### 5.4 规则管理 / 模型设置

两页都改为：

- 顶部页头 + 简要说明
- 下面直接进入主列表/主表单

原因：

- 这两页的主工作对象会持续增长
- 说明文本固定且较少，不应该持续切割空间

### 5.5 帮助中心

当前帮助页只是左半边少量静态文本，不足以承担“中心”角色。

重排后改成完整单栏文档页，内容分成：

- 工作流程
- 模型配置
- 规则编写
- 结果阅读

每一节使用更像说明文档的节奏，而不是几张孤立的小卡片漂在左侧。

### 5.6 评审详情

保留双栏，但调整为非对称结构：

- 左侧更宽，专注原文连续阅读
- 右侧较窄，承接问题清单与问题详情

同时优化原文排版：

- 压缩 block 间距
- 让段落行高与块间距更接近真实正文阅读节奏
- heading、paragraph、list item 的上下留白做差异化，而不是统一拉大

## 6. 导航状态设计

顶部导航需要新增当前页态：

- 当前页链接使用明显不同的背景或描边
- 同时保留 hover 态
- 当前页文字颜色和边框对比度明显高于普通态

实现方式建议使用路由 pathname 判断，而不是手写每页传参。

## 7. CSS 与布局策略

### 7.1 新的共享布局语义

建议引入更明确的布局 class，而不是让 `grid-main` 同时服务所有页面：

- `page-stack`
  用于绝大多数单栏业务页
- `page-header`
  统一标题区
- `page-inline-help`
  极短帮助提示或链接
- `review-layout`
  专供详情页的非对称双栏

### 7.2 间距策略

- 页级区块用较大的 section 间距
- 同一内容流中的正文块用更紧凑的 block 间距
- 说明文本与主内容之间保持距离，但说明本身不做大面积卡片化

### 7.3 响应式策略

- 业务页在桌面端和移动端都优先单栏
- 详情页在宽屏保留非对称双栏，小屏自动折叠
- 导航在小屏下保留 wrap，但当前态样式仍清晰

## 8. 验收标准

- 评审列表不再出现大面积右侧说明区。
- 新建评审、规则管理、模型设置不再使用 50/50 左右布局。
- 帮助页成为完整单栏页面，不再只占左半边。
- 顶部导航在当前页面有清晰高亮。
- 首页不再出现“设计原则/页面理念”式文本。
- 评审详情的原文段落间距明显收紧，阅读节奏更接近正文。

## 9. 风险与处理

### 9.1 风险：页面从“双栏感”变成“太空”

处理：

- 用更合理的主内容最大宽度约束单栏页
- 保留小型统计和短提示，但不再让它们形成独立大侧栏

### 9.2 风险：帮助内容重新变长

处理：

- 帮助中心承接长说明
- 业务页只允许一句级别提示，不再重新复制整段帮助文案

### 9.3 风险：详情页改动影响原有交互

处理：

- 只调整栅格和间距，不改问题点击、原文定位和高亮逻辑
- 通过现有 smoke test 和手动点击链路回归验证
