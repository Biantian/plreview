# 规则管理与评审结果页重构设计

## 1. 目标

本轮改造解决两类核心问题：

- 规则管理页信息密度过低，所有规则默认展开，导致页面过长、维护成本高
- 评审结果页只保留了扁平段落结构，丢失了标题、列表与文档层级，同时问题信息在左右两侧重复

## 2. 规则管理页方案

- 默认展示紧凑规则列表
- 每条规则仅展示名称、分类、默认严重级别、启用状态与更新时间
- 点击“编辑”后再展开单条规则的内联编辑表单
- “新增规则”改为按钮触发，点击后展开创建表单
- 整体压缩卡片间距和内边距，提高扫描效率

## 3. 文档结构升级方案

- 引入 `DocumentBlock` 作为新的文档展示与评审基元
- 每个块包含：
  - `blockIndex`
  - `blockType`
  - `text`
  - `level`
  - `listKind`
  - `charStart`
  - `charEnd`
- 旧的 `DocumentParagraph` 保留作为兼容层，避免旧数据立即失效

## 4. 文档解析策略

- `docx`：从 `mammoth` 的 HTML 输出中提取标题、正文与列表项
- `md`：识别 Markdown 标题与有序/无序列表
- `txt`：基于空行和轻量启发式识别标题与正文块

## 5. 评审链路升级

- Prompt 从“按段落评审”升级为“按文档块评审”
- 模型返回的标注主键从 `paragraphIndex` 迁移为 `blockIndex`
- 注释数据同时保留 `paragraphIndex` 兼容字段

## 6. 结果页交互

- 左侧展示结构化原文，只承担上下文与命中位置展示
- 块顶部仅保留数字索引，不再显示“段落 x”
- 右侧问题清单成为主阅读入口
- 点击右侧问题项后：
  - 左侧对应内容块高亮
  - 右侧下方展开当前问题详情
- 左侧不再重复完整问题描述，只显示命中状态和规则标签
