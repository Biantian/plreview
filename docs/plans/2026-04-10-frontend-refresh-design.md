# 策划案评审系统前端改版设计

## 1. 改版目标

本次改版聚焦两个问题：

- 提升品牌感和展示感，让首页与工作页不再只是功能卡片的集合
- 修复当前界面在表单、双栏和长文本场景下的越界与联动适配问题

## 2. 设计方向

整体方向采用“精品工作台型”：

- 保留后台工具的效率感
- 增强品牌识别与首屏气质
- 首页优先强调“开始新评审”这一主操作

## 3. 视觉系统

- 基础色调从偏米色 MVP 面板升级为更干净的浅底工作台体系
- 主品牌色沿用青绿色，强化系统感和稳定感
- 辅助强调色采用暖橙色，只用于 CTA、局部强调与高亮
- 面板采用高透浅玻璃感，但保证文字与边框对比足够稳定

## 4. 布局策略

- 首页重构为品牌主区 + 工作台概览双区结构
- 新建评审页改为任务启动台，按“任务设置 / 文档导入 / 规则选择”分段
- 规则管理页弱化长表单压迫感，突出规则库与编辑区域分层
- 评审详情页增强问题清单、报告正文与原文段落的联动感

## 5. 响应式与稳定性要求

- 所有栅格子项都必须允许缩小，不得因默认最小内容宽度撑破父容器
- 输入框、文本域、上传控件、列表项在窄屏下自动回落为单列
- 长文本支持换行，避免规则描述、报告内容和文件名挤破边界
- 右侧信息栏仅在宽屏使用粘性定位，窄屏自动回到正常流

## 6. 交互原则

- CTA 优先级明确，主操作显著，次操作克制
- Hover 只做轻微位移和边框反馈，不做破坏布局的动画
- 所有焦点态必须可见，保留键盘操作可达性
- 尊重 `prefers-reduced-motion`
